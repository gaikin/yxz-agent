# AI 技能索引（WARM）

更新时间：2026-06-03

## 使用方式

本文件按需读取。仅在需要选择文档、前端、子进程、脚本 DSL 或测试策略时读取。这里的“技能”不是 Codex 外部技能包，而是本项目内的协作能力分类。

## 文档设计技能

适用场景：

- 更新正式设计文档。
- 梳理架构、术语、流程或脚本方案。
- 将历史 archive 内容整理成正式文档。

检查项：

- 先读 `docs/formal-docs-index.md`。
- 术语必须对齐 `docs/terminology.md`。
- 正文避免旧术语 DCF、Runtime、Channel。
- archive 只作背景，不作为当前约束。
- 不确定的产品含义和架构边界先确认。

## 子进程服务技能

适用场景：

- 修改 `subprocess/service/`。
- 修改定时任务、事件接入、通道服务、轻量持久化。

检查项：

- 子进程不承担 MCP 工具能力。
- 子进程不上传任务记录。
- 子进程只保留轻量状态和触发摘要。
- 跨窗体通信语义优先使用共享协议类型。
- 修改调度能力后优先运行相关 Node 测试。

## 前端窗体技能

适用场景：

- 修改 `webapp/src/`。
- 修改主窗体、任务子窗体、确认弹窗、展示组件或状态管理。

检查项：

- 展示层不直接调用营小助智能体服务、开阳基座或开阳 MCP。
- 展示层通过用户指令和运行事件与执行层交互。
- 状态管理优先沿用 Zustand store。
- UI 风格优先沿用现有 Ant Design、styled-components 和 app theme。
- 重要前端改动后运行 `npm run build:webapp`。

## 脚本 DSL 技能

适用场景：

- 修改事件触发任务脚本设计、制作规范或实现。
- 处理变量、输出、条件、循环、执行记录和静态校验。

检查项：

- 先读 `docs/event-task-script-execution-design.md`。
- 脚本是 JSON DSL，不是 JS 代码。
- 系统变量当前仅支持 `$_EVENT`。
- 工具结果不自动进入变量上下文，必须显式声明 `output`。
- 条件和循环表达式使用 JSON Logic 对象。
- `beforeDelayMs` 可取消，且中止时返回 `USER_CANCELED`。

## 测试与验证技能

适用场景：

- 修改核心服务、协议、调度、执行层或共享类型。

检查项：

- 优先运行最小相关测试。
- 影响跨层协议时运行 `npm test`。
- 影响前端构建时运行 `npm run build:webapp`。
- 影响整体打包时运行 `npm run build`。
- 若测试失败来自既有脏工作区或无关迁移，记录清楚失败点。
