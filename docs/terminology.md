# 营小助术语规范

更新时间：2026-06-03

## 1. 文档目的

本文档用于统一营小助项目中的产品、架构、前端、子进程、任务执行和协议术语，避免不同文档中混用中英文或多个近义词导致设计歧义。

原则：

- 文档正文优先使用规范中文术语。
- 英文或代码字段仅在首次说明、协议字段或文件路径中出现。
- 产品语义和代码字段分开描述，不强行要求协议字段立即改名。
- 当一个词既可表示产品概念又可表示技术资源时，必须拆开命名。

## 2. 核心架构术语

| 规范术语 | 可接受括注 | 含义 | 避免用法 |
| --- | --- | --- | --- |
| 子进程 | resident subprocess | 常驻在宿主环境中的轻量基础设施层，负责触发、唤起、宿主适配和轻量持久化 | DCF 大脑、完整运行时、子进程 Agent |
| 业务窗体 | business window | 承载具体业务交互、执行层和展示层的窗体统称 | 前端页面、UI、窗口容器 |
| 主窗体 | main window | 承载人工对话类能力的主业务窗体 | 对话子窗体、助手页、聊天页 |
| 任务子窗体 | task window | 承载定时任务和事件触发任务的业务子窗体 | 定时任务子窗体、事件任务子窗体、自动化窗口 |
| 确认弹窗 | confirmation popup | 轻量弹窗，只负责展示待确认项并收集确认或忽略操作 | 执行窗口、任务窗口 |
| 执行层 | execution layer | 位于业务窗体内，负责任务调度、MCP 工具调用、步骤推进、结果上报和人工接管处理 | Agent Runtime、智能体运行时、执行器 |
| 展示层 | presentation layer | 位于业务窗体内，负责 UI 展示、用户输入、状态呈现、确认和中止入口 | UI 层、页面层 |
| 平台接入层 | host adapter | 接入开阳基座能力，包括宿主 API、窗体控制、事件路由和窗体间通信 | 宿主适配层、宿主逻辑、控制器逻辑 |

## 3. 运行边界术语

| 规范术语 | 含义 |
| --- | --- |
| 常驻能力 | 子进程在没有业务窗体打开时仍需保留的能力，例如定时触发、事件接入、窗口唤起、轻量持久化 |
| 窗体能力 | 必须依赖业务窗体交互和生命周期的能力，例如任务调度、MCP 工具调用、工具结果回显 |
| 轻量持久化 | 子进程保存的授权状态、任务启用状态、待确认项、触发摘要等小型状态 |
| 窗体临时状态 | 业务窗体内的消息流、步骤状态、工具结果卡片、正在执行的任务队列等状态 |
| 窗体销毁 | 用户关闭承载执行层的业务窗体后，当前一次任务、队列和窗体临时状态全部销毁 |

## 4. 会话与执行术语

| 规范术语 | 代码或英文对应 | 含义 | 说明 |
| --- | --- | --- | --- |
| 业务会话 | Conversation | 用户与营小助智能体的一段业务对话边界 | 文档中优先使用“业务会话” |
| 会话标识 | `sessionId` | 协议和代码中的业务会话标识字段 | 短期不改协议字段 |
| 一次任务 | Run | 一次用户消息、定时任务或事件触发任务启动后的完整执行过程 | 可对应 `runId` |
| 一次任务标识 | `runId` | 协议和代码中的一次任务标识字段 | 不直接写成“运行 ID” |
| 任务步骤 | Step | 一次任务中的单个步骤，例如打开菜单、读取页面结构、执行页面命令 | 可对应 `stepId` |
| 任务步骤标识 | `stepId` | 协议和代码中的任务步骤标识字段 | 仅用于技术定位 |
| 消息 | Message | 业务会话里的用户消息、营小助智能体消息或系统提示 | 避免和事件、通知混用 |
| 工具调用 | Tool Call | 调用 MCP 工具或其他本地工具的一次动作 | 不等同于任务步骤，任务步骤可以包含工具调用 |
| 工具结果 | Tool Result | 一次工具调用返回的结果 | 可用于展示、上报或进入下一步 |

## 5. 任务术语

| 规范术语 | 含义 | 说明 |
| --- | --- | --- |
| 定时任务 | 基于计划时间触发的任务 | 由子进程常驻触发，进入任务子窗体执行 |
| 事件触发任务 | 基于外部业务事件触发的任务 | 已确认与定时任务合并进入任务子窗体 |
| 待确认任务项 | pending execution item | 定时任务或事件触发任务到来后，等待用户确认是否执行的单个项目 | 确认弹窗展示的单项 |
| 待确认任务概览 | pending overview | 当前待确认任务项的列表、数量和更新时间 | 确认弹窗只消费概览 |
| 任务摘要 | execution summary | 子进程可保留的轻量结果，例如成功、失败、忽略、完成时间 | 不包含完整消息流和完整任务步骤 |
| 任务记录 | execution record | 可用于审计、回放和排查的一次任务完整或半完整记录 | 由业务窗体执行层直接上传营小助智能体服务 |
| 触发源 | trigger | 让子进程唤起任务子窗体的来源，例如定时任务、事件触发任务 | 不单独使用触发事件 |
| 上下文 | payload / context | 任务启动时携带的数据，例如定时任务上下文、事件触发任务上下文 | 不直接使用 payload |
| 队列 | queue | 任务子窗体内部串行处理多个待执行项时使用的等待结构 | 例如任务队列 |

## 6. MCP 与工具术语

| 规范术语 | 含义 | 说明 |
| --- | --- | --- |
| MCP 工具能力 | 通过 MCP 调用本地工具或页面自动化能力 | 文档中可保留 MCP 缩写 |
| MCP 会话 | MCP Session | MCP 通信层技术资源，不是业务会话 |
| 页面自动化工具 | 例如 `openMenu`、`executePageCommands` | 具体工具名保留代码写法 |
| 人工接管 | 用户或业务页面要求停止自动化并转人工处理 | 必须进入业务会话或一次任务状态 |

调用 MCP 工具时涉及的 JSON-RPC、SSE 等连接实现不单独定义正式产品术语，只在技术实现说明中按需描述。

## 7. 状态术语

| 规范术语 | 含义 |
| --- | --- |
| 未开始 | 任务或一次任务尚未开始 |
| 等待确认 | 已触发但等待用户确认 |
| 排队中 | 已确认但尚未开始执行 |
| 执行中 | 正在运行智能体调度或工具调用 |
| 已完成 | 执行正常结束 |
| 已失败 | 执行异常结束 |
| 已忽略 | 用户忽略待确认执行项 |
| 已中止 | 用户主动中止当前执行 |
| 已接管 | 当前执行进入人工接管状态 |

代码枚举可以继续使用英文，例如 `pending`、`running`、`completed`。文档正文使用中文状态。

## 8. 通信术语

| 规范术语 | 含义 |
| --- | --- |
| 开阳基座通信 | 主窗体、任务子窗体、确认弹窗与子进程之间交换子进程指令和子进程通知的通信能力 |
| 子进程指令 | 主窗体、任务子窗体、确认弹窗或平台发给子进程的操作请求 |
| 子进程通知 | 子进程发给主窗体、任务子窗体或确认弹窗的状态通知 |
| 用户指令 | 展示层发给执行层的用户操作，例如发送消息、确认执行、中止任务 |
| 运行事件 | 执行层在一次任务运行过程中产生的标准事件，用于驱动展示层更新、任务摘要回写、任务记录上报等 |
| 运行事件分发 | 执行层产生运行事件后通知展示层不同组件的分发机制 |

## 9. 模块与能力术语

| 规范术语 | 含义 |
| --- | --- |
| 状态管理 | 负责维护某一类前端状态，并向展示层和执行层提供读取与更新能力 |
| 服务 | 承担一组稳定业务能力的模块，例如定时触发服务、窗体控制服务 |
| 接入层 | 接入外部系统或平台能力的模块 |
| 平台接入层 | 接入开阳基座能力的接入层 |
| 定时触发器 | 子进程中负责基于计划时间产生定时任务触发源的能力 |
| 事件接入器 | 子进程中负责接入外部业务事件并转成任务触发源的能力 |
| 窗体控制 | 子进程负责的窗体打开、唤起、关闭和初始化参数传递能力 |
| 窗体控制服务 | 实现窗体控制能力的服务 |
| 配置 | 系统运行所需的配置，例如定时任务配置、事件接入配置、平台配置 |
| 持久化 | 将授权状态、任务启用状态、待确认任务项等状态保存下来 |
| 缓存 | 临时保存工具列表、运行中间状态等数据，不等同于持久化 |
| 日志 | 日志和埋点统一使用的术语，不再区分 log 与 telemetry |
| 错误 | 可预期的业务或运行失败，例如授权失败、工具调用失败、任务执行失败 |
| 异常 | 非预期的程序问题或程序缺陷 |
| 授权 | 用户或系统完成某项能力的允许动作或授权状态 |
| 权限 | 某个主体是否具备访问、执行或操作某项资源的资格 |

## 10. 外部对象术语

| 规范术语 | 含义 |
| --- | --- |
| 营小助智能体 | 本项目中的智能体能力和产品对象 |
| 营小助智能体服务 | 远端负责业务会话、智能体编排、工具结果接收的服务，原 Agent Gateway |
| 开阳基座 | 提供宿主能力、开阳基座通信、窗体和子进程承载能力 |
| 开阳 MCP | 提供 MCP 工具能力，例如打开菜单、页面自动化、资源读取 |
| 开阳事件订阅 | 开阳侧事件订阅能力，首次可括注 `eventHook` |
| 技能 | Skill，用于描述可执行的业务能力 |
| 脚本 | Script，用于描述结构化执行脚本 |

正式文档中应尽量避免泛称“后端”或“服务端”，优先写明确对象：营小助智能体服务、开阳基座、开阳 MCP。

## 12. 当前已确认术语

- 常驻基础设施统一称为“子进程”，不再使用“薄子进程”。
- 人工对话承载窗体统一称为“主窗体”。
- 定时任务和事件触发任务合并使用“任务子窗体”。
- 业务窗体内部拆分为“执行层”和“展示层”。
- 子进程不承担 MCP 工具能力。
- 关闭承载执行层的业务窗体时，称为“窗体销毁”。
- 所有任务记录都由业务窗体执行层直接上传营小助智能体服务；子进程不参与任务记录上传链路。
- 任务记录上传失败不阻塞一次任务完成，上传失败状态、日志和必要错误摘要由业务窗体执行层处理。
- 不引入“后台任务”作为正式术语；正式设计暂时没有后台执行方式。
- 不引入“领域事件”，统一使用“运行事件”。
- 不把 `Channel` 和 `Runtime` 作为正式术语。
- 正式文档使用“营小助智能体”，代码和接口名中可保留 Agent。
